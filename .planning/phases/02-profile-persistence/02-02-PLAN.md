        return -1;
    }

**Change 3 — Replace SaveConfig and LoadConfig bodies:**

Remove the bodies of `SaveConfig` and `LoadConfig` (lines ~278-314) and replace with:

```cpp
void SaveConfig(GameProfile* profile) {
    // Binding values are now stored in the profile's JSON file.
    // Font preference continues to be written to settings.ini.
    wchar_t localFontName[FONT_NAME_BUFFER];
    {
        std::lock_guard<std::mutex> lock(g_configMutex);
        StringCchCopy(localFontName, ARRAYSIZE(localFontName), g_fontName);
    }
    WritePrivateProfileString(L"UI", L"FontName", localFontName, CONFIG_FILE);
    SaveProfileBindings(profile);
}

void LoadConfig(GameProfile* profile) {
    // Binding values loaded from JSON (with settings.ini fallback for migration).
    // Font preference loaded from settings.ini.
    LoadProfileBindings(profile);
}
```

Note: `LoadProfileBindings` already handles the mutex internally (under `g_configMutex`) and reads the font from settings.ini into `g_fontName`. The `LoadConfig` wrapper is kept as a call site so all existing callers (`OnGameSelected` at line 748) require no change.

No other changes to CustomControlZ.cpp — `g_gameProfileCount` usages (lines 805, 869, 893, 927) continue to work because they now read the runtime-populated global.

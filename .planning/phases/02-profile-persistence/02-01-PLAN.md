<task type="auto">
  <name>Task 2: Create ProfileLoader.h — parser, loader, saver, migrator</name>
  <files>CustomControlZ/ProfileLoader.h</files>
  <action>
Create `CustomControlZ/ProfileLoader.h`. This is the sole new C++ file in Phase 2. It must be self-contained (include GameProfiles.h, BehaviorEngine.h is pulled in transitively), use only Win32 API + C++20 stdlib, and contain no global state of its own — it populates the caller's `g_gameProfiles` / `g_gameProfileCount`.

**Ownership model:** A static `ProfileStorage` object (or a module-level `std::vector`) owns all heap-allocated `GameProfile` objects and their `wchar_t*` string buffers. The pointers in `GameProfile` fields point into these buffers. Since `GameProfile.id`, `displayName`, etc. are `const wchar_t*`, they must point to heap memory that outlives the loader call.

**Public API (the only symbols CustomControlZ.cpp needs to call):**

```cpp
// Discover and load all *.json from the profiles\ folder next to the exe.
// Populates outProfiles[0..outCount-1] with heap-allocated GameProfile*.
// Caller owns nothing — ProfileLoader owns the memory for app lifetime.
// Returns false and shows MessageBox if no profiles found.
bool LoadProfilesFromFolder(GameProfile**& outProfiles, int& outCount);

// Save updated currentVk values back to the JSON file for this profile.
// Reads the existing JSON file, replaces all "currentVk": N values for the
// profile's bindings, writes to a .tmp file, then MoveFileExW to original.
// profileJsonPath is stored in the profile's user-data slot (see below).
void SaveProfileBindings(GameProfile* profile);

// Load currentVk values from the JSON file. Falls back to settings.ini
// GetPrivateProfileInt for any binding where currentVk == defaultVk (migration).
// Writes migrated values back to JSON immediately.
void LoadProfileBindings(GameProfile* profile);
```

**Implementation plan (write in this order):**

1. `GetExeFolder()` — `GetModuleFileNameW` + strip filename, returns `std::wstring` with trailing backslash.

2. Helper: `ReadFileToString(path)` — `CreateFileW` + `GetFileSizeEx` + `ReadFile` + `CloseHandle`, returns `std::string` (UTF-8). Returns empty on error.

3. Helper: `Utf8ToWide(s)` — `MultiByteToWideChar(CP_UTF8, ...)` → `std::wstring`.

4. Helper: `AllocWstr(ws)` — allocates `new wchar_t[ws.size()+1]`, copies, returns raw `wchar_t*`. Stored in a companion `std::vector<wchar_t*>` owned by the loader for cleanup (though for this app they live forever).

5. **Minimal JSON parser helpers** (all operate on `std::string`):
   - `FindStringValue(json, key)` → `std::string`. Finds `"key": "value"` or `"key": null`. Returns empty for null/missing.
   - `FindIntValue(json, key, defaultVal)` → `int`. Finds `"key": 123`.
   - `FindUInt32Value(json, key, defaultVal)` → `DWORD`. For wheelDelta (may exceed INT_MAX). Use `strtoul`.
   - `ExtractObject(json, key)` → `std::string`. Returns content of `"key": { ... }` (brace-balanced).
   - `ExtractArrayElements(json, key)` → `std::vector<std::string>`. Returns each `{ ... }` element of `"key": [ ... ]`.

6. **BehaviorType parser:**
   ```cpp
   BehaviorType ParseBehaviorType(const std::string& s) {
       if (s == "EdgeTrigger") return BehaviorType::EdgeTrigger;
       if (s == "LongPress")   return BehaviorType::LongPress;
       if (s == "WheelToKey")  return BehaviorType::WheelToKey;
       return BehaviorType::HoldToToggle;
   }
   ```

7. **ParseProfile(fileContent, jsonPath)** — builds a heap-allocated `GameProfile*`:
   - Parse top-level string fields (id, displayName, processName1, processName2, tipActive, tipIdle, settingsTitle).
   - Set `iniSection = id` (same pointer — safe, both point to same heap buffer).
   - processName2: if `FindStringValue(...)` returns empty string AND the raw JSON contains `"processName2": null` or has no processName2 key → set `profile->processName2 = nullptr`. Otherwise allocate normally.
   - Parse theme: `ExtractObject(json, "theme")` → parse each color field with `FindIntValue` (decimal COLORREF). Assign directly as `COLORREF`.
   - Parse bindings: `ExtractArrayElements(json, "bindings")` → for each element, parse iniKey, label, defaultVk, currentVk, behaviorType, outputVk, longOutputVk (default 0), thresholdMs (default 400), durationMs (default 50), wheelDelta (default 120).
   - Set `profile->logicFn = GenericLogicThreadFn`.
   - Store the jsonPath as a side-channel for SaveProfileBindings to use (see note below).

8. **Storing jsonPath per profile:** `GameProfile` has no jsonPath field. Solution: maintain a `static std::map<GameProfile*, std::wstring> g_profileJsonPaths` inside ProfileLoader.h (or inside an anonymous namespace). `SaveProfileBindings` and `LoadProfileBindings` look up the path from this map.

9. **LoadProfilesFromFolder(outProfiles, outCount):**
   - Call `GetExeFolder() + L"profiles\\"` to get folder path.
   - `FindFirstFileW(folder + L"*.json", &fd)` — loop with `FindNextFileW`, always call `FindClose`.
   - For each file: read with `ReadFileToString`, parse with `ParseProfile`.
   - If 0 profiles found: `MessageBox(nullptr, L"No profile files found in profiles\\ folder next to the exe...", L"CustomControlZ — No Profiles", MB_OK | MB_ICONERROR)`, return false.
   - Allocate `GameProfile** arr = new GameProfile*[count]`, fill from parsed profiles vector, set outProfiles/outCount, return true.

10. **SaveProfileBindings(profile):**
    - Look up `jsonPath` from `g_profileJsonPaths`.
    - Read existing JSON file with `ReadFileToString`.
    - For each binding i: replace the i-th occurrence of `"currentVk": N` within the bindings array. Strategy: find each binding object in sequence (use `ExtractArrayElements` positions), replace the `"currentVk"` value within each. Alternatively: do a simple sequential replacement pass — find `"currentVk":` in order of occurrence, replace the number after it with `profile->bindings[i].currentVk`. This is safe because the schema is fixed and currentVk only appears in binding objects.
    - Write to `jsonPath + L".tmp"` via `CreateFileW` + `WriteFile` + `CloseHandle`.
    - `MoveFileExW(tmpPath, jsonPath, MOVEFILE_REPLACE_EXISTING)`.

11. **LoadProfileBindings(profile):**
    - Look up `jsonPath` from `g_profileJsonPaths`.
    - Read JSON file, extract bindings array, for each binding parse its `currentVk` field.
    - If `currentVk == profile->bindings[i].defaultVk` (i.e., not user-customised), fall back to `GetPrivateProfileInt(profile->iniSection, profile->bindings[i].iniKey, profile->bindings[i].defaultVk, L".\\settings.ini")`. If the ini value differs from the default, use it and call `SaveProfileBindings(profile)` immediately to migrate it to JSON.
    - Under `g_configMutex`: set `profile->bindings[i].currentVk` for each binding.
    - Also read font from settings.ini (unchanged behaviour for PROF-04): `GetPrivateProfileString(L"UI", L"FontName", ...)` — copy to `g_fontName`. The font stays in settings.ini.

**Anti-patterns to avoid (from research):**
- Do NOT return local `wchar_t[]` stack arrays as `const wchar_t*` fields of GameProfile — all string fields must be heap-allocated.
- Do NOT set `processName2 = L""` when absent — it must be `nullptr`.
- Do NOT forget `FindClose` on every code path after `FindFirstFileW` succeeds.
- Do NOT use `_wtoi` on wheelDelta — it is DWORD and may exceed INT_MAX; use `strtoul(..., 10)`.
- Do NOT write binding currentVk values to settings.ini — only write font to settings.ini.
